## Objetivo

Construir una aplicacion web de administracion para bar/restaurante, responsive (web + tablets), con roles diferenciados y 6 modulos funcionales.

***

## Arquitectura General


## Estructura de Roles

| Rol      | Acceso                                           |
| -------- | ------------------------------------------------ |
| `admin`  | Todo: caja, reportes, stock, menu, configuracion |
| `mozo`   | Solo sus mesas, tomar pedidos, ver carta         |
| `cocina` | Solo ver comandas pendientes (modo solo lectura) |

* El admin inicia sesion con usuario/contraseña

* Los mozos se identifican por un **ID de dispositivo** (PIN o codigo de mozo) sin password compleja

* La pantalla de cocina es una URL protegida con token fijo (o PIN)

## Modulos y Funcionalidades

### 1. Gestion de Mesas y Reservas

* Mapa visual de mesas (drag-and-drop o grid configurable)

* Estados: `libre`, `ocupada`, `reservada`, `en cuenta`

* Declarar cantidad de personas

* Reservas con fecha, hora, nombre, cantidad de personas, numero de contacto (opcional)

* Asignación de mozo a mesa

  <br />

### 2. Carta / Menu Digital

* Categorias (entradas, platos, bebidas, postres, etc.)

* Productos con nombre, descripcion, precio, imagen, disponibilidad

* Activar/desactivar items sin eliminarlos

### 3. Pedidos y Comandas

* El mozo selecciona mesa, agrega productos y envia comanda

* La cocina ve las comandas en tiempo real (WebSocket)

* Estados de comanda: `pendiente`, `en preparacion`, `listo`, `entregado`

* Posibilidad de agregar productos adicionales a una mesa abierta

### 4. Caja y Facturacion

* Ver cuenta de cada mesa (items + precios)

* Aplicar descuentos o propinas

* Medios de pago: efectivo, tarjeta, voucher de descuento, otros

* Emision de ticket/recibo (PDF imprimible)

* Cierre de caja diario

### 5. Control de Stock / Inventario

* Listado de ingredientes/insumos con stock actual

* Movimientos: entrada, salida manual, consumo automatico por pedido (opcional)

* Alertas de stock minimo

### 6. Reportes y Estadisticas

* Ventas del dia / semana / mes

* Productos mas vendidos

* Ingresos por medio de pago

* Movimientos de caja

* Exportacion a CSV/PDF/xlsx

## Stack Tecnologico

| Capa          | Tecnologia                                  |
| ------------- | ------------------------------------------- |
| Frontend      | React 18 + Vite + TailwindCSS + React Query |
| Backend       | Node.js + Express + `pg` (node-postgres)    |
| Base de datos | PostgreSQL 16                               |
| Auth          | JWT (access token) + PIN para mozos         |
| Tiempo real   | Socket.io (comandas y estado de mesas)      |
| PDF/Tickets   | `pdfkit` o `react-pdf`                      |
| Dev local     | Docker Compose (PostgreSQL + pgAdmin)       |

***

## Pasos de Implementacion

1. **Estructura base** - monorepo, Docker Compose, `.env`, scripts de migracion SQL
2. **Backend core** - Express, conexion PostgreSQL, migraciones, auth JWT + PIN
3. **API por modulo** - rutas, controladores y modelos para cada modulo
4. **WebSocket** - eventos de pedidos y mesas en tiempo real
5. **Frontend base** - Vite + Tailwind, router, AuthContext, layout por rol
6. **Paginas por modulo** - Mesas, Menu, Pedidos, Caja, Stock, Reportes
7. **Vista Cocina** - pantalla simplificada solo lectura
8. **Panel Mozo** - flujo optimizado para tablet
9. **Generacion de tickets PDF**
10. **Testing y ajustes responsive**
